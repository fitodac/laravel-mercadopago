## Publish the config file

Publishing the config file is optional:

```bash
php artisan vendor:publish --tag=mercadopago-config
```

Published file:

```php
<?php

declare(strict_types=1);

use Fitodac\LaravelMercadoPago\Support\EnvCredentialResolver;

return [
    'access_token' => env('MERCADOPAGO_ACCESS_TOKEN'),
    'public_key' => env('MERCADOPAGO_PUBLIC_KEY'),
    'webhook_secret' => env('MERCADOPAGO_WEBHOOK_SECRET'),
    'route_prefix' => env('MERCADOPAGO_ROUTE_PREFIX', 'api/mercadopago'),
    'enable_demo_routes' => (bool) env('MERCADOPAGO_ENABLE_DEMO_ROUTES', true),
    'runtime_environment' => env('MERCADOPAGO_RUNTIME_ENVIRONMENT'),
    'credential_resolver' => EnvCredentialResolver::class,
];
```

## Environment variables

| Variable | Required | Default | Purpose |
| --- | --- | --- | --- |
| `MERCADOPAGO_ACCESS_TOKEN` | Yes | none | SDK authentication for API calls |
| `MERCADOPAGO_PUBLIC_KEY` | No | none | Frontend SDK integrations such as card tokenization |
| `MERCADOPAGO_WEBHOOK_SECRET` | Recommended | none | HMAC validation for webhooks |
| `MERCADOPAGO_ROUTE_PREFIX` | No | `api/mercadopago` | Prefix for webhook and demo routes |
| `MERCADOPAGO_ENABLE_DEMO_ROUTES` | No | `true` | Enables demo endpoints in `local` and `testing` |
| `MERCADOPAGO_RUNTIME_ENVIRONMENT` | No | auto | Forces SDK runtime to `local` or `server` |

## Runtime behavior

If `MERCADOPAGO_RUNTIME_ENVIRONMENT` is empty, the package configures the SDK as:

- `local` when `app()->environment()` is `local` or `testing`
- `server` in all other environments

## Demo route rules

The middleware `EnsureDemoRoutesEnabled` returns `404` unless both conditions are true:

- `MERCADOPAGO_ENABLE_DEMO_ROUTES=true`
- the app environment is `local` or `testing`

The webhook endpoint ignores this middleware and stays active in every environment.

## Custom credential resolver

By default, the package uses `EnvCredentialResolver`, which reads values from Laravel configuration.

You can replace it with your own resolver class:

```php
'credential_resolver' => App\Support\MercadoPago\TenantCredentialResolver::class,
```

Your resolver must implement:

```php
use Fitodac\LaravelMercadoPago\Contracts\CredentialResolverInterface;
use Fitodac\LaravelMercadoPago\DTO\MercadoPagoCredentials;

final class TenantCredentialResolver implements CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials
    {
        return new MercadoPagoCredentials(
            accessToken: 'APP_USR-token',
            publicKey: 'APP_USR-public-key',
            webhookSecret: 'secret',
        );
    }
}
```

Use this only when your project needs per-tenant, per-store, or secret-manager based credential lookup.

## Example setups

### Local development

```ini
APP_ENV=local
MERCADOPAGO_ACCESS_TOKEN=TEST-your-access-token
MERCADOPAGO_PUBLIC_KEY=TEST-your-public-key
MERCADOPAGO_WEBHOOK_SECRET=test-secret
MERCADOPAGO_ENABLE_DEMO_ROUTES=true
```

### Production

```ini
APP_ENV=production
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-access-token
MERCADOPAGO_PUBLIC_KEY=APP_USR-your-public-key
MERCADOPAGO_WEBHOOK_SECRET=production-secret
MERCADOPAGO_ENABLE_DEMO_ROUTES=false
MERCADOPAGO_RUNTIME_ENVIRONMENT=server
```

## Verify configuration

After changing `.env`, clear config cache if needed:

```bash
php artisan config:clear
```

Then call:

```bash
curl http://localhost:8000/api/mercadopago/health
```

## Security notes

- Never store real credentials in committed files
- Always configure `MERCADOPAGO_WEBHOOK_SECRET` in production
- Keep demo routes disabled outside `local` and `testing`
