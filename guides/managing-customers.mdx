## Customer payload

`CreateCustomerRequest` supports:

- `email` required
- `first_name` optional
- `last_name` optional
- `phone` optional
- `identification` optional
- `default_address` optional
- `metadata` optional

## Card payload

`StoreCustomerCardRequest` requires:

- `token`

## Service example

```php
use Fitodac\LaravelMercadoPago\Services\CardService;
use Fitodac\LaravelMercadoPago\Services\CustomerService;

$customer = $customerService->create([
    'email' => 'buyer@example.com',
    'first_name' => 'Jane',
    'last_name' => 'Doe',
]);

$card = $cardService->create((string) data_get($customer, 'id'), [
    'token' => 'card-token',
]);
```

## Demo endpoints

```text
POST /api/mercadopago/customers
GET /api/mercadopago/customers/{customerId}
POST /api/mercadopago/customers/{customerId}/cards
DELETE /api/mercadopago/customers/{customerId}/cards/{cardId}
```

## Use cases

- save customer data before direct payments
- attach tokenized cards for future charges
- fetch customer state before running support or retry flows
