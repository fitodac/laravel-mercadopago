## Supported input

`CreatePaymentRequest` requires:

- `transaction_amount`
- `token`
- `description`
- `installments`
- `payment_method_id`
- `payer.email`

Optional fields:

- `issuer_id`
- `payer.identification`
- `metadata`
- `external_reference`
- `notification_url`

## Service example

```php
use Fitodac\LaravelMercadoPago\Services\PaymentService;

$payment = $paymentService->create([
    'transaction_amount' => 150.75,
    'token' => 'card-token',
    'description' => 'Order #1001',
    'installments' => 1,
    'payment_method_id' => 'visa',
    'payer' => [
        'email' => 'buyer@example.com',
    ],
    'external_reference' => 'order-1001',
]);
```

## Demo endpoints

```text
POST /api/mercadopago/payments
GET /api/mercadopago/payments/{paymentId}
POST /api/mercadopago/payments/{paymentId}/refunds
```

## Recommended production flow

1. tokenize card data on the frontend with Mercado Pago tools
2. send only the token and business-safe fields to your backend
3. create the payment through `PaymentService`
4. persist your own order state
5. confirm the final payment state through webhooks
