## Always-on endpoint

The package always exposes:

```text
POST /api/mercadopago/webhooks
```

This route does not depend on demo-route settings.

## What the package does

`WebhookService`:

- reads the request payload
- validates `x-signature` when a secret is configured
- returns `acknowledged`, `validated`, `topic`, `resource`, and `payload`

## What your app still needs to do

The package does not persist webhook events or update your business records.

Typical app flow:

1. receive the webhook
2. confirm the resource id and topic
3. fetch or reconcile the related payment/order in your app
4. update your business state idempotently

## Local test example

```bash
curl --request POST \
  --url http://localhost:8000/api/mercadopago/webhooks \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "payment",
    "data": {
      "id": "1234567890"
    }
  }'
```

## Production recommendations

- configure `MERCADOPAGO_WEBHOOK_SECRET`
- expose the endpoint over HTTPS
- treat webhook handling as idempotent
- verify business ownership before marking orders as paid
