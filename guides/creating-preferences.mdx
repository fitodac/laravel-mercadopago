## Valid payload

`CreatePreferenceRequest` accepts:

- `items` required
- `payer` optional
- `back_urls` optional
- `metadata` optional
- `notification_url` optional
- `external_reference` optional
- `expires` optional
- `expiration_date_from` optional
- `expiration_date_to` optional

## Service example

```php
use Fitodac\LaravelMercadoPago\Services\PreferenceService;

$preference = $preferenceService->create([
    'items' => [
        [
            'title' => 'Pro plan',
            'quantity' => 1,
            'unit_price' => 49.90,
        ],
    ],
    'payer' => [
        'email' => 'buyer@example.com',
    ],
    'external_reference' => 'subscription-42',
]);
```

## Demo endpoint

```text
POST /api/mercadopago/preferences
```

## Recommended fields

- `external_reference` so you can match Mercado Pago events back to your order
- `notification_url` so Mercado Pago can notify your app
- `back_urls` so the user can return to your application after checkout

## Response notes

The package returns the SDK response normalized inside the standard envelope. The most common fields you will use are:

- `id`
- `init_point`
- `sandbox_init_point`
