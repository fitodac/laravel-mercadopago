## Refund behavior

`RefundService` supports two modes:

- full refund when `amount` is omitted
- partial refund when `amount` is present

## Full refund

```php
$refund = $refundService->create($paymentId);
```

## Partial refund

```php
$refund = $refundService->create($paymentId, [
    'amount' => 25.00,
]);
```

## Demo endpoint

```text
POST /api/mercadopago/payments/{paymentId}/refunds
```

Payload:

```json
{
  "amount": 25.00
}
```

## Recommended business checks

Before calling `RefundService`, verify in your own app:

- the order belongs to the correct account
- the payment is eligible for refund
- the amount does not exceed the refundable balance
