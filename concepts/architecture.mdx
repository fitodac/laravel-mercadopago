## Package building blocks

The package is centered around four layers:

1. `LaravelMercadoPagoServiceProvider`
2. HTTP layer for demo endpoints
3. service classes that wrap Mercado Pago SDK clients
4. support classes for credentials, SDK configuration, and response normalization

## Service provider responsibilities

`LaravelMercadoPagoServiceProvider` does three things:

- merges `config/mercadopago.php`
- binds `CredentialResolverInterface`
- registers singleton services and routes

Registered singletons:

- `MercadoPagoClientFactory`
- `SdkHttpClient`
- `PreferenceService`
- `PaymentService`
- `RefundService`
- `CustomerService`
- `CardService`
- `PaymentMethodService`
- `WebhookService`
- `TestUserService`

## Route model

The package loads routes from `routes/api.php` under:

```php
Route::prefix((string) config('mercadopago.route_prefix', 'api/mercadopago'))
    ->name('mercadopago.')
```

Routes are split into two groups:

- webhook route: always available
- demo routes: protected by `mercadopago.demo`

## Demo route gate

`EnsureDemoRoutesEnabled` aborts with `404` unless:

- `mercadopago.enable_demo_routes` is `true`
- the app environment is `local` or `testing`

This keeps demo tooling out of production, even if the routes are still registered.

## Request flow

Typical flow for a demo endpoint:

1. a `FormRequest` validates the incoming payload
2. a controller calls a package service
3. the service uses `MercadoPagoClientFactory`
4. the factory resolves credentials and configures the SDK
5. `AbstractApiController` wraps the response as `{ ok, data, meta }`

## Error mapping

`AbstractApiController` maps exceptions into HTTP responses:

| Exception type | Status |
| --- | --- |
| `MPApiException` | SDK status code |
| `MercadoPagoConfigurationException` | `422` |
| `InvalidWebhookSignatureException` | `401` |
| `HttpExceptionInterface` | framework status code |
| Any other exception | `500` |

## Runtime environment resolution

`MercadoPagoClientFactory` sets the SDK runtime environment from:

- `MERCADOPAGO_RUNTIME_ENVIRONMENT` when present
- otherwise `local` for `local/testing`
- otherwise `server`

## Webhook path

`WebhookService` handles webhook requests by:

- reading the request payload
- resolving `webhookSecret`
- validating `x-signature` when a secret is configured
- returning `acknowledged`, `validated`, `topic`, `resource`, and `payload`
