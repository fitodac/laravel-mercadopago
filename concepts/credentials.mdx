## Credential types

| Credential | Required | Use |
| --- | --- | --- |
| `accessToken` | Yes | Authenticates Mercado Pago API calls |
| `publicKey` | No | Frontend SDK integrations such as card tokenization |
| `webhookSecret` | Recommended | Validates webhook signatures |

<Warning>
  `accessToken` is sensitive server-side data. Never expose it in frontend code
  or commit it to the repository.
</Warning>

## `MercadoPagoCredentials` DTO

Resolved credentials are represented by:

```php
final readonly class MercadoPagoCredentials
{
    public function __construct(
        public string $accessToken,
        public ?string $publicKey = null,
        public ?string $webhookSecret = null,
    ) {}
}
```

## `CredentialResolverInterface`

The package resolves credentials through:

```php
interface CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials;
}
```

This lets you keep the package generic while changing where credentials come from.

## Default resolver: `EnvCredentialResolver`

By default, the package uses `EnvCredentialResolver`, which:

- reads credentials from Laravel config
- throws `MercadoPagoConfigurationException` when the access token is missing
- converts empty `public_key` and `webhook_secret` values to `null`

## Custom resolver example

```php
use App\Models\Store;
use Fitodac\LaravelMercadoPago\Contracts\CredentialResolverInterface;
use Fitodac\LaravelMercadoPago\DTO\MercadoPagoCredentials;

final class StoreCredentialResolver implements CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials
    {
        $store = Store::query()->firstOrFail();

        return new MercadoPagoCredentials(
            accessToken: $store->mercadopago_access_token,
            publicKey: $store->mercadopago_public_key,
            webhookSecret: $store->mercadopago_webhook_secret,
        );
    }
}
```

Then point the config to it:

```php
'credential_resolver' => App\Support\MercadoPago\StoreCredentialResolver::class,
```

## Recommended rules

- keep `MERCADOPAGO_ACCESS_TOKEN` required in every environment
- set `MERCADOPAGO_WEBHOOK_SECRET` in production
- use a custom resolver only when your app actually needs dynamic credentials
