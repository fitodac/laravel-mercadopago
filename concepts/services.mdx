## Why the service layer matters

The package is designed so your application can consume Mercado Pago through injected services instead of coupling business logic to controllers or SDK internals.

## Available methods

| Service | Public methods |
| --- | --- |
| `PreferenceService` | `create(array $payload)` |
| `PaymentService` | `create(array $payload)`, `get(string $paymentId)` |
| `RefundService` | `create(string $paymentId, array $payload = [])` |
| `CustomerService` | `create(array $payload)`, `get(string $customerId)` |
| `CardService` | `create(string $customerId, array $payload)`, `delete(string $customerId, string $cardId)` |
| `PaymentMethodService` | `all()` |
| `WebhookService` | `handle(Request $request)` |
| `TestUserService` | `create(array $payload)` |

## Injecting a service

Use the services directly in your own controller:

```php
use Fitodac\LaravelMercadoPago\Services\PreferenceService;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;

final class CheckoutController
{
    public function store(
        Request $request,
        PreferenceService $preferenceService,
    ): JsonResponse {
        $payload = $request->validate([
            'items' => ['required', 'array', 'min:1'],
            'items.*.title' => ['required', 'string'],
            'items.*.quantity' => ['required', 'integer', 'min:1'],
            'items.*.unit_price' => ['required', 'numeric', 'min:0'],
        ]);

        $preference = $preferenceService->create($payload);

        return response()->json([
            'id' => data_get($preference, 'id'),
            'init_point' => data_get($preference, 'init_point'),
        ], 201);
    }
}
```

## When to use services vs demo endpoints

Use services when:

- you are building production checkout flows
- you need your own auth or rate limiting
- you want to persist orders or payments
- you need custom request validation or business rules

Use demo endpoints when:

- you are testing package wiring locally
- you need quick smoke tests for SDK connectivity
- you want simple integration examples

## Method behavior details

- `RefundService::create()` calls `refund()` when `amount` exists and `refundTotal()` otherwise
- `CardService` uses the customer card client and supports add/delete operations only
- `TestUserService` uses `SdkHttpClient` because test user creation is not exposed through the same high-level client API
- `WebhookService` validates signatures only when both `MERCADOPAGO_WEBHOOK_SECRET` and `x-signature` are present
