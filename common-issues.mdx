## Configuration Issues

<AccordionGroup>
  <Accordion title="Mercado Pago access token is not configured">
    **Error Message:**
    ```
    Mercado Pago access token is not configured.
    ```

    **Solution:**
    1. Add the access token to your `.env` file:
       ```bash
       MERCADOPAGO_ACCESS_TOKEN=your_access_token_here
       ```
    2. Clear the configuration cache:
       ```bash
       php artisan config:clear
       ```
  </Accordion>

  <Accordion title="Package is installed but routes don't appear">
    ```bash
    composer dump-autoload
    php artisan package:discover
    php artisan route:clear
    php artisan route:list --name=mercadopago
    ```
  </Accordion>
</AccordionGroup>

## Demo Routes Issues

<AccordionGroup>
  <Accordion title="404 error on demo routes">
    Demo routes only work when **both** conditions are met:
    - `MERCADOPAGO_ENABLE_DEMO_ROUTES=true`
    - Application environment is `local` or `testing`

    ```bash .env
    APP_ENV=local
    MERCADOPAGO_ENABLE_DEMO_ROUTES=true
    ```

    Then clear configuration cache:
    ```bash
    php artisan config:clear
    ```
  </Accordion>

  <Accordion title="Route prefix conflicts">
    Change the route prefix in your `.env`:
    ```bash
    MERCADOPAGO_ROUTE_PREFIX=api/payments/mercadopago
    ```
  </Accordion>
</AccordionGroup>

## Webhook Issues

<AccordionGroup>
  <Accordion title="Webhook returns 401 Unauthorized">
    ```json
    {"ok": false, "message": "Mercado Pago webhook signature is invalid."}
    ```

    Check:
    1. `MERCADOPAGO_WEBHOOK_SECRET` matches Mercado Pago dashboard
    2. Request includes `x-signature` and `x-request-id` headers
    3. URL includes `data.id` in query string or payload
  </Accordion>

  <Accordion title="Webhook not receiving notifications">
    1. Webhook URL must use HTTPS in production
    2. For local development, use a tunnel service:
       ```bash
       ngrok http 8000
       ```
    3. Register the public URL in Mercado Pago dashboard
  </Accordion>
</AccordionGroup>

## SDK and API Errors

<AccordionGroup>
  <Accordion title="Mercado Pago PHP SDK is not installed">
    ```bash
    composer require mercadopago/dx-php:^3.8
    composer dump-autoload
    php artisan config:clear
    ```
  </Accordion>

  <Accordion title="422 Unprocessable Entity from Mercado Pago">
    **Common causes:**
    - Invalid payload structure
    - Invalid email format
    - Invalid amount
    - Invalid payment method for your country

    Validate your payload against the examples in the guides.
  </Accordion>

  <Accordion title="401 Unauthorized from Mercado Pago API">
    Invalid or expired access token. Verify credentials in MercadoPago dashboard and update your `.env`:
    ```bash
    MERCADOPAGO_ACCESS_TOKEN=APP-1234567890-VALID-TOKEN
    ```
  </Accordion>
</AccordionGroup>

## Testing Issues

<AccordionGroup>
  <Accordion title="Card token is invalid or expired">
    Card tokens generated by MercadoPago.js expire after a short time. Generate the token immediately before creating the payment. Don't store tokens for later use.
  </Accordion>

  <Accordion title="Test users endpoint not working">
    Ensure demo routes are enabled and use a valid `site_id`:
    ```json
    {"site_id": "MLA", "description": "Test user for QA"}
    ```

    Valid site IDs: `MLA`, `MLB`, `MLC`, `MLM`, `MLU`, `MCO`, `MPE`
  </Accordion>
</AccordionGroup>
