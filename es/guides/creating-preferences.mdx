---
title: Crear preferences
description: Construí checkout preferences de Mercado Pago con PreferenceService o con el endpoint demo.
---

## Payload válido

`CreatePreferenceRequest` acepta:

- `items` obligatorio
- `payer` opcional
- `back_urls` opcional
- `metadata` opcional
- `notification_url` opcional
- `external_reference` opcional
- `expires` opcional
- `expiration_date_from` opcional
- `expiration_date_to` opcional

## Ejemplo con service

```php
use Fitodac\LaravelMercadoPago\Services\PreferenceService;

$preference = $preferenceService->create([
    'items' => [
        [
            'title' => 'Pro plan',
            'quantity' => 1,
            'unit_price' => 49.90,
        ],
    ],
    'payer' => [
        'email' => 'buyer@example.com',
    ],
    'external_reference' => 'subscription-42',
]);
```

## Endpoint demo

```text
POST /api/mercadopago/preferences
```

## Campos recomendados

- `external_reference` para vincular eventos de Mercado Pago con tu order
- `notification_url` para que Mercado Pago notifique a tu app
- `back_urls` para que el usuario vuelva a tu aplicación después del checkout

## Notas sobre la respuesta

El paquete devuelve la respuesta del SDK normalizada dentro del envelope estándar. Los campos más usados suelen ser:

- `id`
- `init_point`
- `sandbox_init_point`
