Las preferencias de pago son el pilar fundamental de la experiencia clásica de "Checkout Pro". Se encargan de dictaminar qué artículos van a cobrarse, hacia dónde va a volver el usuario una vez que el cobro finalice, y de qué forma tu sistema va a enterarse de todo esto.

## Entendiendo a las Preferencias

Una preferencia siempre aloja:

- **Items**: Los productos, impuestos o servicios canjeables.
- **Información del Comprador (Payer)**: El correo, nombre y otra metadata opcional del cliente.
- **URLs de Retorno (Back URLs)**: Sitios adonde rebotará el cliente tras abonar con éxito, fallo o queda en suspenso.
- **URL de Notificación (Notification URL)**: El destino remoto donde recaerán las llamadas de sistema del Webhook POST.
- **Referencia Externa (External reference)**: El campo para atarlo a tus pedidos de e-commerce internos (EJ: ID 551).

## Creación básica

<CodeGroup>
```php Mediante PreferenceService
<?php

use Fitodac\LaravelMercadoPago\Services\PreferenceService;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;

final class CheckoutPreferenceController
{
    public function store(
        Request $request,
        PreferenceService $preferenceService
    ): JsonResponse {
        $payload = $request->validate([
            'items' => ['required', 'array', 'min:1'],
            'items.*.title' => ['required', 'string'],
            'items.*.quantity' => ['required', 'integer', 'min:1'],
            'items.*.unit_price' => ['required', 'numeric', 'min:0'],
        ]);

        $preference = $preferenceService->create($payload);

        return response()->json([
            'id' => data_get($preference, 'id'),
            'init_point' => data_get($preference, 'init_point'),
            'sandbox_init_point' => data_get($preference, 'sandbox_init_point'),
        ], 201);
    }
}
```

```bash Mediante ruta demo
curl --request POST \
  --url http://localhost:8000/api/mercadopago/preferences \
  --header 'Content-Type: application/json' \
  --data '{
    "items": [
      {
        "title": "Suscripción Premium",
        "quantity": 1,
        "unit_price": 99.99
      }
    ]
  }'
```
</CodeGroup>

## Entendiendo a las URLs

### Back URLs

Se usan *estrictamente para mover al usuario de vuelta* a tu página final una vez procesen.

- **success**: Tarjeta de crédito/débito o dinero en cuenta aprobado al instante.
- **pending**: Eligieron transferir o abonar en redes de cobro en efectivo.
- **failure**: Salieron del modal prematuramente o la tarjeta fue denegada por fondos insuficientes.

<Warning>
  Las Back URLs solo tienen peso en la Experiencia de Usuario (UI/UX). Nunca construyas la lógica de dar de alta o despachar un producto dependiendo de si el usuario cayó o no en la ruta de `/success`. ¡Usa Webhooks para eso!
</Warning>

### URLs de Notificación (Webhooks)

El puerto al cual los servidores estáticos de MP atacan de forma invisible de fondo por POST para confirmar el estado final real:

<Tip>
  Tu proveedor `mercadopago.route_prefix` de Laravel MercadoPago te inyecta de por sí la ruta segura y optimizada `/api/mercadopago/webhooks`.
</Tip>

## Arquitectura de Respuestas

Toda creación correcta de preferencias devuelve el siguiente diccionario armado:

```json
{
  "ok": true,
  "data": {
    "id": "1234567890-abc123def456",
    "init_point": "https://www.mercadopago.com/checkout/v1/redirect?pref_id=...",
    "sandbox_init_point": "https://sandbox.mercadopago.com/checkout/v1/redirect?pref_id=...",
    "date_created": "2026-03-10T10:30:00.000-03:00",
    "items": [...],
    "payer": {...},
    "back_urls": {...},
    "external_reference": "ORDER-12345"
  },
  "meta": []
}
```

### Campos clave de vital importancia

- **id**: Representación alfanumérica persistente de tu Preferencia (se extingue sola en su vencimiento si existiese).
- **init_point**: El enlace donde rediriges para que operen en modalidad producción.
- **sandbox_init_point**: Ídem al anterior pero inofensivo.

## Patrones de uso diario

### Ligar de manera rígida Preferencias a Órdenes

```php
$orden = Pedido::create([...]);

$preference = $preferenceService->create([
    'items' => $orden->articulos->map(fn($item) => [
        'title' => $item->nombre,
        'quantity' => $item->cantidad,
        'unit_price' => $item->precio,
    ])->toArray(),
    'external_reference' => "ORDEN-{$orden->id}",
    'notification_url' => route('mercadopago.webhooks.store'),
]);
```

### Metadatos Inyectados

Guardá pequeñas piezas de correlación útiles para cuando recibas la respuesta por webhook:

```php
$preference = $preferenceService->create([
    'items' => [...],
    'metadata' => [
        'usuario_registrado_id' => auth()->id(),
        'origen_de_venta' => 'aplicacion_movil_ios',
    ],
]);
```

### Forzando Vencimientos de Preferencias

Bloqueá especuladores de precios fijándoles el momento de expiración exacta:

```php
$preference = $preferenceService->create([
    'items' => [...],
    'expires' => true,
    'expiration_date_from' => now()->toIso8601String(),
    'expiration_date_to' => now()->addHours(6)->toIso8601String(), // Sólo 6 horas de chances de pagarla
]);
```
