## Habilitar demo routes

Usá:

```ini
APP_ENV=local
MERCADOPAGO_ENABLE_DEMO_ROUTES=true
```

## Demo endpoints disponibles para pruebas

| Método | Ruta |
| --- | --- |
| `GET` | `/api/mercadopago/health` |
| `GET` | `/api/mercadopago/payment-methods` |
| `POST` | `/api/mercadopago/preferences` |
| `POST` | `/api/mercadopago/payments` |
| `GET` | `/api/mercadopago/payments/{paymentId}` |
| `POST` | `/api/mercadopago/payments/{paymentId}/refunds` |
| `POST` | `/api/mercadopago/customers` |
| `GET` | `/api/mercadopago/customers/{customerId}` |
| `POST` | `/api/mercadopago/customers/{customerId}/cards` |
| `DELETE` | `/api/mercadopago/customers/{customerId}/cards/{cardId}` |
| `POST` | `/api/mercadopago/test-users` |

## Crear un test user

```bash
curl --request POST \
  --url http://localhost:8000/api/mercadopago/test-users \
  --header 'Content-Type: application/json' \
  --data '{
    "site_id": "MLA",
    "description": "Local QA user"
  }'
```

## Secuencia práctica de prueba

1. llamar a `/health`
2. listar payment methods
3. crear una preference
4. crear o consultar un customer
5. probar la recepción de webhooks

## Notas

- los demo endpoints son solo para `local` y `testing`
- la ruta de webhook puede probarse en cualquier entorno
- probar direct card payments sigue requiriendo flujos válidos de tokenización frontend
