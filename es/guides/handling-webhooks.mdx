title: Manejar webhooks
description: Recibí notificaciones de Mercado Pago de forma segura y conectalas con el estado de tu aplicación.
---

## Endpoint siempre activo

El paquete siempre expone:

```text
POST /api/mercadopago/webhooks
```

Esta ruta no depende de la configuración de demo routes.

## Qué hace el paquete

`WebhookService`:

- lee el payload de la request
- valida `x-signature` cuando hay un secret configurado
- devuelve `acknowledged`, `validated`, `topic`, `resource` y `payload`

## Qué sigue siendo responsabilidad de tu app

El paquete no persiste eventos de webhook ni actualiza tus registros de negocio.

Flujo típico de app:

1. recibir el webhook
2. confirmar el resource id y el topic
3. buscar o reconciliar el payment/order relacionado en tu app
4. actualizar el estado de negocio de forma idempotente

## Ejemplo de prueba local

```bash
curl --request POST \
  --url http://localhost:8000/api/mercadopago/webhooks \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "payment",
    "data": {
      "id": "1234567890"
    }
  }'
```

## Recomendaciones para producción

- configurá `MERCADOPAGO_WEBHOOK_SECRET`
- exponé el endpoint por HTTPS
- tratá el manejo del webhook como idempotente
- verificá la propiedad de negocio antes de marcar orders como pagadas
