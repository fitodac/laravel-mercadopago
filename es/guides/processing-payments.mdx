## Input soportado

`CreatePaymentRequest` requiere:

- `transaction_amount`
- `token`
- `description`
- `installments`
- `payment_method_id`
- `payer.email`

Campos opcionales:

- `issuer_id`
- `payer.identification`
- `metadata`
- `external_reference`
- `notification_url`

## Ejemplo con service

```php
use Fitodac\LaravelMercadoPago\Services\PaymentService;

$payment = $paymentService->create([
    'transaction_amount' => 150.75,
    'token' => 'card-token',
    'description' => 'Order #1001',
    'installments' => 1,
    'payment_method_id' => 'visa',
    'payer' => [
        'email' => 'buyer@example.com',
    ],
    'external_reference' => 'order-1001',
]);
```

## Endpoints demo

```text
POST /api/mercadopago/payments
GET /api/mercadopago/payments/{paymentId}
POST /api/mercadopago/payments/{paymentId}/refunds
```

## Flujo recomendado en producción

1. tokenizá los datos de la tarjeta en el frontend con herramientas de Mercado Pago
2. enviá a tu backend solo el token y campos seguros de negocio
3. creá el payment con `PaymentService`
4. persistí tu propio estado de order
5. confirmá el estado final del payment mediante webhooks
