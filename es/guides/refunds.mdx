El paquete oficialista provee una vía limpia y ágil de anular y devolver los pagos que ya cruzaron la barrera de tu frontend y fueron sellados como exitosos (`approved`).

## Conceptos a Interiorizar

Puntos de severidad respecto a cómo piensa el sistema sobre tus transacciones:

- **Estrictamente** los pagos que figuran como `approved` admiten un reintegro.
- Está permitido dividir la transacción y devolver plata de manera parcial (fraccionada).
- El Gateway acciona tu pedido al milisegundo, pero el saldo no va a impactar en la tarjeta del cliente o resumen hasta pasados unos días hábiles dependiendo de su banco emisor (3 a 30 días).
- Todo ticket de devolución genera un ID autoincremental contable único.

<Warning>
  El proceso es destructivo y unidireccional. Mercado Pago **jamás** te dejará deshacer ni cobrarle de nuevo automáticamente a una persona a la que acabas de ejecutarle un Refund a su tarjeta.
</Warning>

## Reintegro Íntegro y Total

Restituir a tope el dinero descontado:

<CodeGroup>
```php Mediante inyección RefundService
<?php

use Fitodac\LaravelMercadoPago\Services\RefundService;

final class GeneradorReembolsos
{
    public function deshacerCompraDeBolsa(string $paymentId, RefundService $refund): JsonResponse 
    {
        // Cancelación absoluta - solo le damos un payload vacio
        $ticketReintegro = $refund->create($paymentId);

        return response()->json($ticketReintegro, 201);
    }
}
```

```bash Mediante URL demo
curl --request POST \
  --url http://localhost:8000/api/mercadopago/payments/12345/refunds \
  --header 'Content-Type: application/json' \
  --data '{}'
```
</CodeGroup>

## Devoluciones Fraccionadas

Ideal para cuando el cliente retuvo una prenda pero pidió el dinero de vuelta únicamente del calzado dentro del mismo resumen.

<CodeGroup>
```php RefundService en Parcialidad
public function devolucionFraccionada(string $paymentId, Request $request, RefundService $r): JsonResponse 
{
    $dinero = $request->validate([
        'amount' => ['required', 'numeric', 'min:0.01'],
    ]);

    $ticket = $r->create($paymentId, $dinero);

    return response()->json($ticket, 201);
}
```

```bash Demo
curl --request POST \
  --url http://localhost:8000/api/mercadopago/payments/1234567/refunds \
  --header 'Content-Type: application/json' \
  --data '{
    "amount": 50.00
  }'
```
</CodeGroup>

## Arquitectura de Validaciones recomendadas

### Tope máximo de Devolución

No podés intentar devolver más plata fraccionadamente de lo que valió el ticket entero.

```php
public function procesarMitad(string $pId, Request $req, PaymentService $mp, RefundService $r): JsonResponse 
{
    $factura = $mp->get($pId);

    if ($req->amount > $factura['transaction_amount']) {
        return response()->json([
            'error' => 'Aritmética inválida, pretendés devolver más de lo que entró',
            'maximo' => $factura['transaction_amount'],
        ], 422);
    }

    return response()->json($r->create($pId, ['amount' => $req->amount]));
}
```

### Anulaciones disparadas automáticamente (Eventos de Cancelación de Pedidos)

Cuando tu empleado o operario decida cancelar manual, conectá ese gatillo de Eloquent y pedile el refund interconectando a Mercado Pago por atrás:

```php
public function eliminarPedidoParaSiempre(Order $pedido)
{
    if ($pedido->status === 'pagado' && $pedido->mp_payment_id) {
        $this->motorReintegro->create($pedido->mp_payment_id);
    }
    
    $pedido->update(['status' => 'anulado_devuelto']);
}
```
