## Comportamiento del refund

`RefundService` soporta dos modos:

- full refund cuando omitís `amount`
- partial refund cuando enviás `amount`

## Full refund

```php
$refund = $refundService->create($paymentId);
```

## Partial refund

```php
$refund = $refundService->create($paymentId, [
    'amount' => 25.00,
]);
```

## Endpoint demo

```text
POST /api/mercadopago/payments/{paymentId}/refunds
```

Payload:

```json
{
  "amount": 25.00
}
```

## Validaciones de negocio recomendadas

Antes de llamar a `RefundService`, verificá en tu app:

- que la order pertenezca a la cuenta correcta
- que el payment sea elegible para refund
- que el monto no supere el saldo reembolsable
