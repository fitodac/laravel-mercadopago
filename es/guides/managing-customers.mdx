## Payload de customer

`CreateCustomerRequest` soporta:

- `email` obligatorio
- `first_name` opcional
- `last_name` opcional
- `phone` opcional
- `identification` opcional
- `default_address` opcional
- `metadata` opcional

## Payload de card

`StoreCustomerCardRequest` requiere:

- `token`

## Ejemplo con service

```php
use Fitodac\LaravelMercadoPago\Services\CardService;
use Fitodac\LaravelMercadoPago\Services\CustomerService;

$customer = $customerService->create([
    'email' => 'buyer@example.com',
    'first_name' => 'Jane',
    'last_name' => 'Doe',
]);

$card = $cardService->create((string) data_get($customer, 'id'), [
    'token' => 'card-token',
]);
```

## Endpoints demo

```text
POST /api/mercadopago/customers
GET /api/mercadopago/customers/{customerId}
POST /api/mercadopago/customers/{customerId}/cards
DELETE /api/mercadopago/customers/{customerId}/cards/{cardId}
```

## Casos de uso

- guardar datos de customer antes de direct payments
- asociar tokenized cards para futuros cobros
- consultar el estado de un customer antes de un flujo de soporte o reintento
