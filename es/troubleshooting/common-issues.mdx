---
title: Problemas comunes
description: Resolvé los problemas más frecuentes de setup y ejecución en Laravel MercadoPago.
---

## `Mercado Pago access token is not configured.`

Causa:

- `MERCADOPAGO_ACCESS_TOKEN` falta o está vacío

Solución:

```ini
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-access-token
```

Después limpiá la caché de config:

```bash
php artisan config:clear
```

## Las demo endpoints devuelven `404`

Causa:

- `MERCADOPAGO_ENABLE_DEMO_ROUTES` no está en `true`
- o el entorno de la app no es `local` ni `testing`

Solución:

```ini
APP_ENV=local
MERCADOPAGO_ENABLE_DEMO_ROUTES=true
```

## El webhook devuelve `401`

Causa:

- `MERCADOPAGO_WEBHOOK_SECRET` no coincide
- el header `x-signature` está mal formado
- falta `x-request-id` o `data.id` en la request firmada

Solución:

- verificá el secret en Mercado Pago
- confirmá el formato `ts=...,v1=...`
- confirmá que el payload o query string incluya el resource id

## `Mercado Pago PHP SDK is not installed.`

Causa:

- la instalación del paquete está incompleta
- las dependencias no se instalaron bien en la app host

Solución:

```bash
composer install
composer show mercadopago/dx-php
```

## Las rutas usan el prefijo equivocado

Causa:

- `MERCADOPAGO_ROUTE_PREFIX` cambió pero la caché de config sigue guardando un valor viejo

Solución:

```bash
php artisan config:clear
php artisan route:list --name=mercadopago
```

## El health endpoint muestra `configured: false`

Causa:

- falló la resolución de credenciales

Solución:

- verificá `MERCADOPAGO_ACCESS_TOKEN`
- verificá el `credential_resolver` custom
- confirmá que el resolver devuelva un `accessToken` no vacío
