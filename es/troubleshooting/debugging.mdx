title: Debugging
description: Usá inspección de rutas, revisión de config y smoke tests pequeños para depurar integraciones con Laravel MercadoPago.
---

## Empezá por las rutas

```bash
php artisan route:list --name=mercadopago
```

Revisá que:

- el prefijo coincida con `MERCADOPAGO_ROUTE_PREFIX`
- exista la ruta de webhook
- estén presentes los nombres de demo routes

## Revisá la configuración efectiva

Inspeccioná el entorno y la config publicada:

```bash
php artisan config:clear
php artisan tinker
```

Dentro de Tinker:

```php
config('mercadopago')
```

## Usá el health endpoint

En `local` o `testing`:

```bash
curl http://localhost:8000/api/mercadopago/health
```

Eso confirma:

- resolución del access token
- si existe `publicKey`
- si existe `webhookSecret`
- entorno Laravel actual

## Depurar firmas de webhook

Si falla la validación del webhook:

- registrá el `x-signature` entrante
- registrá el `x-request-id` entrante
- confirmá que el payload incluya `data.id`
- confirmá que el secret coincida con Mercado Pago

## Depurar uso de services en tu app

Cuando falle tu propio controller, probá el service de forma aislada:

```php
app(\Fitodac\LaravelMercadoPago\Services\PreferenceService::class)->create([
    'items' => [
        [
            'title' => 'Debug item',
            'quantity' => 1,
            'unit_price' => 10,
        ],
    ],
]);
```
