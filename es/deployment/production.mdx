## Objetivos de producción

En producción, la configuración del paquete debería:

- usar credenciales reales de Mercado Pago
- mantener las demo routes fuera de servicio
- aceptar webhooks por HTTPS
- validar firmas de webhook

## `.env` recomendado

```ini
APP_ENV=production
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-production-access-token
MERCADOPAGO_PUBLIC_KEY=APP_USR-your-production-public-key
MERCADOPAGO_WEBHOOK_SECRET=your-production-webhook-secret
MERCADOPAGO_ROUTE_PREFIX=api/mercadopago
MERCADOPAGO_ENABLE_DEMO_ROUTES=false
MERCADOPAGO_RUNTIME_ENVIRONMENT=server
```

## Verificaciones obligatorias

1. Confirmá que la webhook URL apunte a tu dominio real con HTTPS.
2. Confirmá que `MERCADOPAGO_ENABLE_DEMO_ROUTES=false`.
3. Confirmá que `MERCADOPAGO_WEBHOOK_SECRET` coincida con el valor en Mercado Pago.
4. Limpiá y regenerá la caché de config después de cambiar secretos.

## Expectativa sobre rutas

Ejecutá:

```bash
php artisan route:list --name=mercadopago
```

La lista sigue mostrando demo routes, pero las requests a esas rutas deberían devolver `404` en producción por `EnsureDemoRoutesEnabled`.

La ruta de webhook debe seguir activa:

```text
POST /api/mercadopago/webhooks
```

## Checklist de despliegue

- [ ] `MERCADOPAGO_ACCESS_TOKEN` está definido
- [ ] `MERCADOPAGO_PUBLIC_KEY` está definido si usás tokenización frontend
- [ ] `MERCADOPAGO_WEBHOOK_SECRET` está definido
- [ ] `MERCADOPAGO_ENABLE_DEMO_ROUTES=false`
- [ ] HTTPS está habilitado
- [ ] la caché de config fue refrescada
- [ ] la webhook URL está configurada en Mercado Pago

## Smoke tests

Después del deploy:

```bash
php artisan config:clear
php artisan route:list --name=mercadopago
curl -X POST https://your-domain.com/api/mercadopago/webhooks -H 'Content-Type: application/json' -d '{}'
```

Esperá que el endpoint de webhook responda, incluso si el payload no sirve todavía para el negocio.
