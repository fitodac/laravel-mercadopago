## Resumen

La clase `PreferenceService` maneja todo el andamiaje para armar pagos delegados mediante checkout gráfico en MP. Permiten definir el carrito de compras a cobrar y a su vez establecen atajos de retorno de éxito para tu cliente.

**Ubicación de archivo**: `src/Services/PreferenceService.php:9`

#### Parámetros Principales

<ParamField path="payload" type="array" required>
  Toda la configuración y data en crudo.
  <Expandable title="Estructura del Payload Analizada">
    <ParamField path="items" type="array" required>
      Diccionario Multidimensional del Carrito.
      - `title` (string, obligatorio): Nombre o título del ítem
      - `quantity` (integer, obligatorio): Cuántos pedís
      - `unit_price` (numeric, obligatorio): El monto cobrable por ítem aislado
      - `currency_id` (string, opcional): Default a moneda local del vendedor
      - `description` (string, opcional): Metadata extensa para adornarlo
    </ParamField>

    <ParamField path="payer" type="array">
      El comprador (Cliente Humano/Jurídico): 
      - `email` (string): Correo 
      - `name` (string): Nombre real
      - `surname` (string): Apellidos
      - `phone` (array): Teléfono formateado
      - `address` (array): Calles de despacho material
    </ParamField>

    <ParamField path="back_urls" type="array">
      Las ubicaciones web de retroceso desde el Checkout Finalizado de MP hacia Tu Sistema: 
      - `success` (string): Ganaste
      - `pending` (string): En Veremos financiero
      - `failure` (string): Perdiste la venta (se arrepintió o no tiene saldo)
    </ParamField>

    <ParamField path="notification_url" type="string">
      Webhook exclusivo para vigilar el comportamiento de este carrito en la sombra.
    </ParamField>

    <ParamField path="external_reference" type="string">
      Nomenclatura propia (Atalo a los ID de `pedidos` en tu DB SQL).
    </ParamField>
  </Expandable>
</ParamField>

#### Uso en la Práctica Cotidiana

<CodeGroup>
```php Mediante Inyección
use Fitodac\LaravelMercadoPago\Services\PreferenceService;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;

final class GeneradorController
{
    public function store(Request $r, PreferenceService $service): JsonResponse 
    {
        $compra = $service->create([
            'items' => [
                ['title' => 'Producto', 'quantity' => 1, 'unit_price' => 5000],
            ],
            'back_urls' => [
                'success' => route('compras.exito'),
                'pending' => route('compras.suspenso'),
                'failure' => route('compras.falla'),
            ],
            'notification_url' => route('mercadopago.webhooks'),
            'external_reference' => 'NUEVO-10',
        ]);

        return response()->json([
            'preference_id' => data_get($compra, 'id'),
            'checkout_url' => data_get($compra, 'init_point'),
        ]);
    }
}
```
