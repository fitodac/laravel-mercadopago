## Resumen

El objeto `CustomerService` abstrae el puente hacia MP y te permite crear y/o descargar a tus legajos los datos sincronizados de un "cliente duradero". Estos perfiles son los únicos que Mercado Pago autoriza para poder amarrarle indefinidamente múltiples y variables tarjetas de crédito y débito post-tokenización.

## Métodos

### create()

Inventa y empuja un nuevo Customer y abre un id público.

```php
public function create(array $payload): mixed
```

#### Rutinas Normales

<CodeGroup>
```php Inyección y Auto-Salvado.
use Fitodac\LaravelMercadoPago\Services\CustomerService;

public function registrarYPegarEnMp(Request $req, CustomerService $service) 
{
    // Creás cliente local MySQL ... etc
    $usuario = User::create([...]);

    // Acto seguido empujás y armás copia en Mercado Pago
    $customerRemoto = $service->create([
        'email' => $usuario->email,
        'first_name' => $usuario->name,
    ]);

    // Pegamento Final de Relaciones
    $usuario->update([
        'mp_customer_id' => data_get($customerRemoto, 'id'),
    ]);

    return "Usuario Creado y Syncronizado PCI";
}
```
</CodeGroup>

### get()

Extrayendo de vuelta y resincronizando en base a la columna esclava local de nuestro proyecto.

```php
public function get(string $customerId): mixed
```

#### Invocando a resincronización

<CodeGroup>
```php Checkeando viabilidad (Existencia de perfil perenne)

$customerGiganteDropeado = app(CustomerService::class)->get($idBaseDeDatosLocal);
```
