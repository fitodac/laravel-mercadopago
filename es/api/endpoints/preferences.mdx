title: Endpoint de preferences
description: Referencia del endpoint demo para crear checkout preferences.
---

## Disponibilidad

Este endpoint solo responde cuando las demo routes están habilitadas y el entorno de la app es `local` o `testing`.

## Ruta

```text
POST /api/mercadopago/preferences
```

## Campos obligatorios

- `items[].title`
- `items[].quantity`
- `items[].unit_price`

## Campos opcionales

- `payer`
- `back_urls`
- `metadata`
- `notification_url`
- `external_reference`
- `expires`
- `expiration_date_from`
- `expiration_date_to`

## Respuesta exitosa

```json
{
  "ok": true,
  "data": {
    "id": "1234567890",
    "init_point": "https://www.mercadopago.com/checkout/v1/redirect?pref_id=..."
  },
  "meta": []
}
```
