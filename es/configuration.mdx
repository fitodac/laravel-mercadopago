## Publicar el archivo de configuración

Publicar el archivo es opcional:

```bash
php artisan vendor:publish --tag=mercadopago-config
```

Archivo publicado:

```php
<?php

declare(strict_types=1);

use Fitodac\LaravelMercadoPago\Support\EnvCredentialResolver;

return [
    'access_token' => env('MERCADOPAGO_ACCESS_TOKEN'),
    'public_key' => env('MERCADOPAGO_PUBLIC_KEY'),
    'webhook_secret' => env('MERCADOPAGO_WEBHOOK_SECRET'),
    'route_prefix' => env('MERCADOPAGO_ROUTE_PREFIX', 'api/mercadopago'),
    'enable_demo_routes' => (bool) env('MERCADOPAGO_ENABLE_DEMO_ROUTES', true),
    'runtime_environment' => env('MERCADOPAGO_RUNTIME_ENVIRONMENT'),
    'credential_resolver' => EnvCredentialResolver::class,
];
```

## Variables de entorno

| Variable | Requerida | Default | Propósito |
| --- | --- | --- | --- |
| `MERCADOPAGO_ACCESS_TOKEN` | Sí | none | Autenticación del SDK para llamadas API |
| `MERCADOPAGO_PUBLIC_KEY` | No | none | Integraciones frontend, como tokenización de tarjetas |
| `MERCADOPAGO_WEBHOOK_SECRET` | Recomendada | none | Validación HMAC de webhooks |
| `MERCADOPAGO_ROUTE_PREFIX` | No | `api/mercadopago` | Prefijo para webhook y demo routes |
| `MERCADOPAGO_ENABLE_DEMO_ROUTES` | No | `true` | Habilita endpoints demo en `local` y `testing` |
| `MERCADOPAGO_RUNTIME_ENVIRONMENT` | No | auto | Fuerza el runtime del SDK a `local` o `server` |

## Comportamiento del runtime

Si `MERCADOPAGO_RUNTIME_ENVIRONMENT` está vacío, el paquete configura el SDK así:

- `local` cuando `app()->environment()` es `local` o `testing`
- `server` en cualquier otro entorno

## Reglas de demo routes

El middleware `EnsureDemoRoutesEnabled` devuelve `404` salvo que se cumplan ambas condiciones:

- `MERCADOPAGO_ENABLE_DEMO_ROUTES=true`
- el entorno de la app sea `local` o `testing`

El endpoint de webhook ignora este middleware y sigue activo en todos los entornos.

## Custom credential resolver

Por default, el paquete usa `EnvCredentialResolver`, que lee valores desde la configuración de Laravel.

Podés reemplazarlo por tu propia clase:

```php
'credential_resolver' => App\Support\MercadoPago\TenantCredentialResolver::class,
```

Tu resolver debe implementar:

```php
use Fitodac\LaravelMercadoPago\Contracts\CredentialResolverInterface;
use Fitodac\LaravelMercadoPago\DTO\MercadoPagoCredentials;

final class TenantCredentialResolver implements CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials
    {
        return new MercadoPagoCredentials(
            accessToken: 'APP_USR-token',
            publicKey: 'APP_USR-public-key',
            webhookSecret: 'secret',
        );
    }
}
```

Usalo solo cuando tu proyecto necesite resolver credenciales por tenant, store o secret manager.

## Ejemplos de setup

### Desarrollo local

```ini
APP_ENV=local
MERCADOPAGO_ACCESS_TOKEN=TEST-your-access-token
MERCADOPAGO_PUBLIC_KEY=TEST-your-public-key
MERCADOPAGO_WEBHOOK_SECRET=test-secret
MERCADOPAGO_ENABLE_DEMO_ROUTES=true
```

### Producción

```ini
APP_ENV=production
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-access-token
MERCADOPAGO_PUBLIC_KEY=APP_USR-your-public-key
MERCADOPAGO_WEBHOOK_SECRET=production-secret
MERCADOPAGO_ENABLE_DEMO_ROUTES=false
MERCADOPAGO_RUNTIME_ENVIRONMENT=server
```

## Verificar configuración

Después de cambiar `.env`, limpiá la caché de config si hace falta:

```bash
php artisan config:clear
```

Luego llamá a:

```bash
curl http://localhost:8000/api/mercadopago/health
```

## Notas de seguridad

- Nunca guardes credenciales reales en archivos versionados
- Configurá siempre `MERCADOPAGO_WEBHOOK_SECRET` en producción
- Dejá las demo routes deshabilitadas fuera de `local` y `testing`
