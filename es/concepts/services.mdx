## Por qué importa la capa de services

El paquete está pensado para que tu aplicación consuma Mercado Pago mediante services inyectados, sin acoplar la lógica de negocio a controllers demo ni a detalles internos del SDK.

## Métodos disponibles

| Service | Métodos públicos |
| --- | --- |
| `PreferenceService` | `create(array $payload)` |
| `PaymentService` | `create(array $payload)`, `get(string $paymentId)` |
| `RefundService` | `create(string $paymentId, array $payload = [])` |
| `CustomerService` | `create(array $payload)`, `get(string $customerId)` |
| `CardService` | `create(string $customerId, array $payload)`, `delete(string $customerId, string $cardId)` |
| `PaymentMethodService` | `all()` |
| `WebhookService` | `handle(Request $request)` |
| `TestUserService` | `create(array $payload)` |

## Inyectar un service

Usá los services directamente en tu propio controller:

```php
use Fitodac\LaravelMercadoPago\Services\PreferenceService;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;

final class CheckoutController
{
    public function store(
        Request $request,
        PreferenceService $preferenceService,
    ): JsonResponse {
        $payload = $request->validate([
            'items' => ['required', 'array', 'min:1'],
            'items.*.title' => ['required', 'string'],
            'items.*.quantity' => ['required', 'integer', 'min:1'],
            'items.*.unit_price' => ['required', 'numeric', 'min:0'],
        ]);

        $preference = $preferenceService->create($payload);

        return response()->json([
            'id' => data_get($preference, 'id'),
            'init_point' => data_get($preference, 'init_point'),
        ], 201);
    }
}
```

## Cuándo usar services y cuándo endpoints demo

Usá services cuando:

- estés armando flujos de producción
- necesites tu propio auth o rate limiting
- quieras persistir orders o payments
- necesites validación o reglas de negocio propias

Usá endpoints demo cuando:

- estés probando el cableado del paquete en local
- necesites smoke tests rápidos de conectividad con el SDK
- quieras ejemplos simples de integración

## Detalles de comportamiento

- `RefundService::create()` llama a `refund()` cuando existe `amount` y a `refundTotal()` si no
- `CardService` usa el customer card client y solo soporta alta y baja de cards
- `TestUserService` usa `SdkHttpClient` porque test user creation no está expuesto por el mismo cliente high-level
- `WebhookService` valida firmas solo cuando existen `MERCADOPAGO_WEBHOOK_SECRET` y `x-signature`
