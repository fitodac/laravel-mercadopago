# Resumen de Servicios

Laravel MercadoPago provee ocho clases de servicios dedicadas exclusivamente, cada una encargada de algún proceso fundamental de la interacción con la API de Mercado Pago. Todos nacen registrados como *Singletons* y podés invocarlos al inyectar su dependencia.

## Arquitectura del Servicio

Los servicios persiguen un patrón de conducta claro:

1. **Dependencia a MercadoPagoClientFactory** - Recurren a esta fábrica local para que interceda un cliente nativo SDK original.
2. **Abstracciones del SDK** - Todos se enfocan en volver a los métodos originales de MP en código local súper amistoso y "Laravelizado".
3. **Type Safety** - Usan y demandan entidades de retornos tipados.

## Servicios Core Básicos

### PreferenceService

**Propósito**: Crear Preferencias de Pago del sistema Checkout Pro de Mercado Pago.

**Ubicación**: `src/Services/PreferenceService.php`

**Casos de uso**:
- Cobros donde se saca al cliente de la plataforma
- Definir métodos de cuotas con re-direccionamientos 

```php
public function create(array $payload): mixed
```

Crea las preferencias y devuelve datos como el `init_point` que necesitas para la url a re-dirigir.

**Ejemplo**:

```php
use Fitodac\LaravelMercadoPago\Services\PreferenceService;

class CheckoutController
{
    public function createCheckout(PreferenceService $service)
    {
        $preference = $service->create([
            'items' => [
                [
                    'title' => 'Suscripción Premium',
                    'quantity' => 1,
                    'unit_price' => 99.99,
                    'currency_id' => 'ARS'
                ]
            ],
            // ... variables complementarias
        ]);

        return redirect($preference->init_point);
    }
}
```

### PaymentService

**Propósito**: Procesar todos aquellos cobros hechos directamente en los componentes propios o Checkout Transparentes.

**Casos de uso**:
- Cobros por tarjeta integrados mediante el SDK de Frontend sin redireccionar la página
- Traer históricos (Get) de pago

**Ejemplos Reales**:

```php
public function process(PaymentService $service, Request $request)
{
    $payment = $service->create([
        'transaction_amount' => 150.00,
        'token' => $request->token, // Generado front-end
        'installments' => 1,
        'payment_method_id' => 'visa',
        'payer' => ['email' => $request->email]
    ]);

    return response()->json([
        'status' => $payment->status,
        'id' => $payment->id
    ]);
}
```

### CustomerService

**Propósito**: Dar de alta o verificar entidades para que MP registre a las personas.

**Ubicación**: `src/Services/CustomerService.php`

Se lo usa mayoritariamente al intentar lograr escenarios del estilo "Pagos En Un Sólo Clic", donde necesitamos de un "Customer" fijo para asignar tarjetas de cuenta.

### CardService

**Propósito**: Registrar métodos de cobros atados a los "Customers" del punto anterior.

**Ejemplo de cobro suscripción (Tokenización)**:

```php
public function destroy(CardService $service, string $cardId)
{
    $customerId = auth()->user()->mp_customer_id;
    $service->delete($customerId, $cardId);

    return response()->json(['message' => 'Tarjeta extirpada y eliminada para el comprador local.']);
}
```

<Tip>
 No podés adjuntar tarjetas si el Customer anterior falla de crearse. Acordate de ejecutar esa función de precondición primero.
</Tip>

### RefundService

**Propósito**: Emitir devoluciones totales y parciales sobre facturas de pagos generadas.

Si pasás el monto vacío devuelve en total el dinero, si aplicás un `amount` a la variable se toma por reembolso fragmentario.

```php
//Total
$refund = $this->refundService->create($paymentId);

//Parcial 
$refund = $this->refundService->create($paymentId, ['amount' => 500]);
```

### PaymentMethodService

Te enlista las distintas formas viables y activas que tenés de recibir cobros sobre un País predefinido. 

Se lo utiliza frecuentemente en los front-end modernos para consultar de forma asíncrona los logotipos y promociones sin recargar página (`$this->paymentMethodService->all()`).

### WebhookService

**Propósito**: Validador principal al momento de la llegada de alertas desde el mundo exterior de MP.

Devolverá un arreglo saneado y un control que acusa o deniega la verificación `HMAC`:

```php
$webhook = $this->webhookService->handle($request);

if ($webhook['validated']) {
    // Fue comprobada la seguridad
}
```
