## Tipos de credenciales

| Credencial | Requerida | Uso |
| --- | --- | --- |
| `accessToken` | Sí | Autentica llamadas API a Mercado Pago |
| `publicKey` | No | Integraciones frontend como tokenización de tarjetas |
| `webhookSecret` | Recomendada | Valida firmas de webhook |

<Warning>
  `accessToken` es dato sensible del lado servidor. Nunca lo expongas en código
  frontend ni lo subas al repositorio.
</Warning>

## DTO `MercadoPagoCredentials`

Las credenciales resueltas se representan así:

```php
final readonly class MercadoPagoCredentials
{
    public function __construct(
        public string $accessToken,
        public ?string $publicKey = null,
        public ?string $webhookSecret = null,
    ) {}
}
```

## `CredentialResolverInterface`

El paquete resuelve credenciales mediante:

```php
interface CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials;
}
```

Eso permite mantener el paquete genérico mientras cambiás la fuente de credenciales.

## Resolver por default: `EnvCredentialResolver`

Por default, el paquete usa `EnvCredentialResolver`, que:

- lee credenciales desde la config de Laravel
- arroja `MercadoPagoConfigurationException` si falta el access token
- convierte `public_key` y `webhook_secret` vacíos a `null`

## Ejemplo de custom resolver

```php
use App\Models\Store;
use Fitodac\LaravelMercadoPago\Contracts\CredentialResolverInterface;
use Fitodac\LaravelMercadoPago\DTO\MercadoPagoCredentials;

final class StoreCredentialResolver implements CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials
    {
        $store = Store::query()->firstOrFail();

        return new MercadoPagoCredentials(
            accessToken: $store->mercadopago_access_token,
            publicKey: $store->mercadopago_public_key,
            webhookSecret: $store->mercadopago_webhook_secret,
        );
    }
}
```

Después apuntá la config a esa clase:

```php
'credential_resolver' => App\Support\MercadoPago\StoreCredentialResolver::class,
```

## Reglas recomendadas

- mantené `MERCADOPAGO_ACCESS_TOKEN` como obligatorio en todos los entornos
- configurá `MERCADOPAGO_WEBHOOK_SECRET` en producción
- usá un custom resolver solo cuando tu app realmente necesite credenciales dinámicas
