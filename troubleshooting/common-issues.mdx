## `Mercado Pago access token is not configured.`

Cause:

- `MERCADOPAGO_ACCESS_TOKEN` is missing or empty

Fix:

```ini
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-access-token
```

Then clear config cache:

```bash
php artisan config:clear
```

## Demo endpoints return `404`

Cause:

- `MERCADOPAGO_ENABLE_DEMO_ROUTES` is not `true`
- or the app environment is not `local` or `testing`

Fix:

```ini
APP_ENV=local
MERCADOPAGO_ENABLE_DEMO_ROUTES=true
```

## Webhook returns `401`

Cause:

- `MERCADOPAGO_WEBHOOK_SECRET` does not match
- `x-signature` header is malformed
- `x-request-id` or `data.id` is missing from the signed request

Fix:

- verify the secret in Mercado Pago
- confirm the header format `ts=...,v1=...`
- confirm the payload or query string includes the resource id

## `Mercado Pago PHP SDK is not installed.`

Cause:

- package installation is incomplete
- dependencies were not installed correctly in the host app

Fix:

```bash
composer install
composer show mercadopago/dx-php
```

## Routes use the wrong prefix

Cause:

- `MERCADOPAGO_ROUTE_PREFIX` changed but config cache still holds an old value

Fix:

```bash
php artisan config:clear
php artisan route:list --name=mercadopago
```

## Health endpoint shows `configured: false`

Cause:

- credential resolution failed

Fix:

- verify `MERCADOPAGO_ACCESS_TOKEN`
- verify custom `credential_resolver` class bindings
- confirm the resolver returns a non-empty `accessToken`
