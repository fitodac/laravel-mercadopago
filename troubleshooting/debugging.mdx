title: Debugging
description: Use route inspection, config checks, and small smoke tests to debug Laravel MercadoPago integrations.
---

## Start with route inspection

```bash
php artisan route:list --name=mercadopago
```

Check:

- prefix matches `MERCADOPAGO_ROUTE_PREFIX`
- webhook route exists
- demo route names are present

## Check effective configuration

Inspect your environment and published config:

```bash
php artisan config:clear
php artisan tinker
```

Inside Tinker:

```php
config('mercadopago')
```

## Use the health endpoint

In `local` or `testing`:

```bash
curl http://localhost:8000/api/mercadopago/health
```

This confirms:

- access token resolution
- whether a `publicKey` exists
- whether a `webhookSecret` exists
- current Laravel environment

## Debug webhook signatures

If webhook validation fails:

- log the incoming `x-signature`
- log the incoming `x-request-id`
- confirm the payload includes `data.id`
- confirm the secret matches Mercado Pago

## Debug service usage in your app

When your own controller fails, test the service in isolation:

```php
app(\Fitodac\LaravelMercadoPago\Services\PreferenceService::class)->create([
    'items' => [
        [
            'title' => 'Debug item',
            'quantity' => 1,
            'unit_price' => 10,
        ],
    ],
]);
```
