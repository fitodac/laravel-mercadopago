## Requirements

Before you install the package, confirm that your project uses:

- PHP `^8.2`
- Laravel `^12.0`
- Composer

## Standard installation

Install the package in any Laravel project:

```bash
composer require fitodac/laravel-mercadopago:^1.0
```

Laravel auto-discovers:

```php
Fitodac\LaravelMercadoPago\LaravelMercadoPagoServiceProvider::class
```

## Local path repository workflow

If the package lives inside the same workspace, add a `path` repository to the host app `composer.json`:

```json
{
  "repositories": [
    {
      "type": "path",
      "url": "plugins/laravel-mercadopago",
      "options": {
        "symlink": true
      }
    }
  ]
}
```

Then install it with the same Composer command:

```bash
composer require fitodac/laravel-mercadopago:^1.0
```

## Optional: publish configuration

You only need this when you want a project-level `config/mercadopago.php` file:

```bash
php artisan vendor:publish --tag=mercadopago-config
```

## Verify package registration

Run:

```bash
php artisan package:discover
php artisan route:list --name=mercadopago
```

You should see route names such as:

- `mercadopago.webhooks.store`
- `mercadopago.health`
- `mercadopago.preferences.store`
- `mercadopago.payments.store`

<Note>
  The demo routes only answer requests when `MERCADOPAGO_ENABLE_DEMO_ROUTES=true`
  and the Laravel environment is `local` or `testing`. The webhook route is
  always registered.
</Note>

## Minimal `.env` setup

Add at least your access token:

```ini
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-access-token
MERCADOPAGO_PUBLIC_KEY=APP_USR-your-public-key
MERCADOPAGO_WEBHOOK_SECRET=your-webhook-secret
MERCADOPAGO_ROUTE_PREFIX=api/mercadopago
MERCADOPAGO_ENABLE_DEMO_ROUTES=true
MERCADOPAGO_RUNTIME_ENVIRONMENT=
```

<Warning>
  Keep real credentials in `.env` or a secret manager. Never commit them to the
  repository.
</Warning>

## First verification request

With demo routes enabled in `local` or `testing`, call:

```bash
curl http://localhost:8000/api/mercadopago/health
```

Expected shape:

```json
{
  "ok": true,
  "data": {
    "configured": true,
    "has_public_key": true,
    "has_webhook_secret": true,
    "environment": "local"
  },
  "meta": []
}
```
