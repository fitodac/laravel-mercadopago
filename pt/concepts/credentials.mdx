## Tipos de credenciais

| Credencial | Obrigatória | Uso |
| --- | --- | --- |
| `accessToken` | Sim | Autentica chamadas de API ao Mercado Pago |
| `publicKey` | Não | Integrações frontend como tokenização de cartões |
| `webhookSecret` | Recomendada | Valida assinaturas de webhook |

<Warning>
  `accessToken` é dado sensível do lado do servidor. Nunca o exponha no frontend
  nem faça commit dele no repositório.
</Warning>

## DTO `MercadoPagoCredentials`

As credenciais resolvidas são representadas assim:

```php
final readonly class MercadoPagoCredentials
{
    public function __construct(
        public string $accessToken,
        public ?string $publicKey = null,
        public ?string $webhookSecret = null,
    ) {}
}
```

## `CredentialResolverInterface`

O pacote resolve credenciais por meio de:

```php
interface CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials;
}
```

Isso permite manter o pacote genérico enquanto você troca a origem das credenciais.

## Resolver default: `EnvCredentialResolver`

Por default, o pacote usa `EnvCredentialResolver`, que:

- lê credenciais da config do Laravel
- lança `MercadoPagoConfigurationException` se o access token estiver ausente
- converte `public_key` e `webhook_secret` vazios em `null`

## Exemplo de custom resolver

```php
use App\Models\Store;
use Fitodac\LaravelMercadoPago\Contracts\CredentialResolverInterface;
use Fitodac\LaravelMercadoPago\DTO\MercadoPagoCredentials;

final class StoreCredentialResolver implements CredentialResolverInterface
{
    public function resolve(): MercadoPagoCredentials
    {
        $store = Store::query()->firstOrFail();

        return new MercadoPagoCredentials(
            accessToken: $store->mercadopago_access_token,
            publicKey: $store->mercadopago_public_key,
            webhookSecret: $store->mercadopago_webhook_secret,
        );
    }
}
```

Depois aponte a config para essa classe:

```php
'credential_resolver' => App\Support\MercadoPago\StoreCredentialResolver::class,
```

## Regras recomendadas

- mantenha `MERCADOPAGO_ACCESS_TOKEN` obrigatório em todos os ambientes
- configure `MERCADOPAGO_WEBHOOK_SECRET` em produção
- use um custom resolver somente quando a sua app realmente precisar de credenciais dinâmicas
