## Production goals

In production, your package setup should:

- use real Mercado Pago credentials
- keep demo routes unavailable
- accept webhooks over HTTPS
- validate webhook signatures

## Recommended `.env`

```ini
APP_ENV=production
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-production-access-token
MERCADOPAGO_PUBLIC_KEY=APP_USR-your-production-public-key
MERCADOPAGO_WEBHOOK_SECRET=your-production-webhook-secret
MERCADOPAGO_ROUTE_PREFIX=api/mercadopago
MERCADOPAGO_ENABLE_DEMO_ROUTES=false
MERCADOPAGO_RUNTIME_ENVIRONMENT=server
```

## Required checks

1. Verify that the webhook URL points to your real HTTPS domain.
2. Confirm that `MERCADOPAGO_ENABLE_DEMO_ROUTES=false`.
3. Confirm that `MERCADOPAGO_WEBHOOK_SECRET` matches the value in Mercado Pago.
4. Clear and rebuild Laravel config cache after changing secrets.

## Route expectations

Run:

```bash
php artisan route:list --name=mercadopago
```

The route list still shows demo routes, but requests to them should return `404` in production because of `EnsureDemoRoutesEnabled`.

The webhook route should remain active:

```text
POST /api/mercadopago/webhooks
```

## Deployment checklist

- [ ] `MERCADOPAGO_ACCESS_TOKEN` is set
- [ ] `MERCADOPAGO_PUBLIC_KEY` is set when frontend tokenization is used
- [ ] `MERCADOPAGO_WEBHOOK_SECRET` is set
- [ ] `MERCADOPAGO_ENABLE_DEMO_ROUTES=false`
- [ ] HTTPS is enabled
- [ ] config cache has been refreshed
- [ ] webhook URL is configured in Mercado Pago

## Smoke tests

After deploy:

```bash
php artisan config:clear
php artisan route:list --name=mercadopago
curl -X POST https://your-domain.com/api/mercadopago/webhooks -H 'Content-Type: application/json' -d '{}'
```

Expect the webhook endpoint to answer, even if the payload is not useful for business processing.
