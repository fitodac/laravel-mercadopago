## Security priorities

For this package, the main security controls are:

- protecting `MERCADOPAGO_ACCESS_TOKEN`
- validating webhooks with `MERCADOPAGO_WEBHOOK_SECRET`
- keeping demo routes out of production traffic
- applying your own auth and authorization to business endpoints

## Webhook signature validation

When `MERCADOPAGO_WEBHOOK_SECRET` is configured and `x-signature` is present, `WebhookService` validates the request.

Validation inputs:

- `x-signature`
- `x-request-id`
- `data.id` from query string or payload

Manifest format:

```text
id:{resourceId};request-id:{requestId};ts:{timestamp};
```

The package computes an HMAC-SHA256 hash and compares it with `v1` from the header.

If validation fails, the controller returns `401`.

## HTTPS

Use HTTPS for every production webhook endpoint.

If your app needs forced HTTPS URLs, you can apply:

```php
public function boot(): void
{
    if ($this->app->environment('production')) {
        \Illuminate\Support\Facades\URL::forceScheme('https');
    }
}
```

## Credential handling

- keep credentials in `.env` or a secret manager
- rotate credentials through environment changes, not code changes
- never expose `accessToken` to the frontend
- only expose `publicKey` where Mercado Pago frontend flows require it

## Demo endpoint exposure

Never treat package demo endpoints as a public business API.

They intentionally do not add:

- auth
- authorization
- throttling
- domain validation
- persistence

## Recommended production pattern

Use your own controller and inject package services:

```php
Route::middleware(['auth:sanctum', 'throttle:10,1'])->group(function () {
    Route::post('/checkout/preferences', CheckoutPreferenceController::class);
});
```
